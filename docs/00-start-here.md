# Start Here

This playbook helps you make product explainer films where the product itself carries the story.

The default output is a clean 1080p film. A subtitle cut can be added later, but the clean cut should work on its own.

## The Core Idea

The product should act while the narrator talks.

Each sentence needs one visual actor:

- A card appears.
- A number counts up.
- A pointer clicks.
- A status changes.
- A panel dims while another becomes active.
- A chat reply streams.
- A row moves from candidate to confirmed.

If nothing changes while a sentence is spoken, the viewer is being asked to listen and stare. That usually feels slow.

## The Production Loop

1. Define the business question.
2. Write a short narration.
3. Split narration into beats.
4. Choose one anchor object or persona to follow.
5. Build or prepare the product screen.
6. Generate or record voiceover.
7. Transcribe voiceover to word timestamps.
8. Drive visual states from the timestamps.
9. Capture frames through a browser.
10. Stitch video and audio.
11. QC with frame grabs, contact sheets, text scans, and timing checks.

## Two Useful Film Types

- App-native explainer: a real product UI is driven frame by frame.
- Hybrid story: generated character moments introduce context, then the film moves into real UI.

Use app-native when the product logic is the hero. Use hybrid when the viewer needs human context, emotion, or a boardroom-style setup.

## First Scene Rule

Do not build the whole film before proving the standard.

Pick one representative beat. Make it excellent. Confirm the visual language, timing, and density. Then roll the system out across the rest of the film.

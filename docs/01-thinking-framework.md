# Thinking Framework

Strong product films start from the decision the audience needs to make, then bring in the features that answer it.

## 1. Start With The Business Question

Bad starting point:

- "Show the dashboard."
- "Show the recommendation panel."
- "Show the workflow."

Better starting point:

- "Can leaders see readiness before it becomes a risk?"
- "Can managers understand why the recommendation was made?"
- "Can employees see a path they trust?"

The product is there to answer the question.

## 2. Make One Object Travel

Pick one anchor object and follow it across the film:

- A skill.
- A risk.
- A customer.
- A workflow item.
- A forecast.
- A recommended action.

The viewer should understand the system because they can track one thing from input to decision to proof.

## 3. Give Every Sentence A Job

For every narration sentence, write:

- What is being said?
- What should the viewer look at?
- What changes on screen?
- What proof appears?

If the sentence has no visual job, cut it or create the visual it deserves.

## 4. Cause Before Effect

Do not let screens magically change.

Show cause:

- A pointer clicks.
- A rule activates.
- An agent status changes.
- A stream begins.
- A row is selected.

Then show effect:

- Output appears.
- Counter changes.
- Badge lands.
- Plan updates.
- Risk cools.

Cause and effect is what makes the product feel real.

## 5. Product Polish Counts As Film Polish

If a screen looks weak on camera, the product probably needs better hierarchy, labels, or state design.

Improve the product screen where possible:

- Rename raw IDs into human labels.
- Add clear status rows.
- Make key numbers visible.
- Add evidence labels.
- Reduce clutter.
- Hide demo-only controls in render mode.

Do not solve weak product hierarchy by putting decorative boxes over it.

# Visual Completeness

Use this checklist to avoid half-built scenes.

## The Three-Layer Rule

Every non-product framework scene should have three layers:

1. Canvas layer: subtle grid, glow, rule, or frame texture.
2. Hero layer: the main object, loop, card, or proof mechanism.
3. Support layer: metrics, evidence rails, labels, trays, or status rows.

If a framework scene has only headline text and one icon, it is not finished.

## Product Scene Density

Real app scenes should get richness from the product itself:

- Meaningful cards.
- Evidence rows.
- Status chips.
- Numbers and bars.
- Audit or proof records.
- Native focus states.

Do not decorate over a weak product screen. Improve the screen hierarchy.

## Empty Quadrant Test

Pause on any frame. Divide it into four quadrants.

If one quadrant is empty, ask:

- Should a support rail live there?
- Should the hero object be repositioned?
- Should the camera crop or scroll be changed?
- Is this a deliberate calm moment?

Calm is fine. Accidental emptiness is not.

## Rich Does Not Mean Busy

Add useful texture:

- A number with a linked bar.
- A small evidence stack.
- A chapter footer.
- A short rule chip.
- A status row.
- A proof tray.

Avoid:

- Decorative blobs.
- Random icons.
- Long paragraphs.
- Competing accent colors.
- Multiple hot rings.

## Minimum Scene Bar

Before a scene is accepted, it should have:

- One clear actor.
- One visual proof.
- One supporting rail or evidence detail.
- One settled end frame.
- No small unreadable text.
- No large accidental blank areas.

# Production Recipes

Use these recipes when planning a specific film.

## Recipe 1: Product Dashboard Film

Best for leader-facing dashboards, analytics, planning tools, or operational views.

Structure:

1. Ask the business question.
2. Show one dashboard metric that matters.
3. Drill into one row or card.
4. Explain the evidence.
5. Show the action or decision.
6. Return to the dashboard with the changed outcome.

Visual actors:

- Metric count-up.
- Row hot-ring.
- Filter chip activation.
- Evidence drawer.
- Status change.

Avoid:

- Touring every chart.
- Letting dashboards sit static.
- Using tiny table text as the main proof.

## Recipe 2: Guided Recommendation Film

Best for products where an assistant explains, drafts, recommends, or reviews.

Structure:

1. Show the user problem.
2. Show the real input context.
3. Click or invoke the assistant.
4. Stream the first answer word by word.
5. Show evidence or citations.
6. Show the user accepting, editing, or rejecting.

Visual actors:

- Pointer click.
- Assistant status chip.
- Typewriter stream with caret.
- Evidence chips.
- Edited field or accepted recommendation.

Avoid:

- Showing a generic chat panel disconnected from product data.
- Streaming long paragraphs.
- Hiding why the assistant made the recommendation.

## Recipe 3: Workflow Proof Film

Best for approval flows, compliance, operations, finance, HR, sales, or support workflows.

Structure:

1. Start with an unresolved item.
2. Show rules or inputs being checked.
3. Show conflict or exception.
4. Show resolution.
5. Show record or audit trail.

Visual actors:

- Checklist ticks.
- Rule chips.
- Conflict warning.
- Approval button.
- Audit row.

Avoid:

- Making the system change state without visible cause.
- Using a fake overlay instead of product state.

## Recipe 4: Hybrid Human + UI Film

Best for launches, board updates, transformation narratives, or sales stories.

Structure:

1. Human setup: why the problem matters.
2. Bridge to product: screen, board, or laptop transition.
3. Real UI proof: product acts.
4. Human or narrator outcome: why it matters.
5. Closing loop: proof artifact or business outcome.

Visual actors:

- Short generated or filmed character clip.
- Match-cut or dip into UI.
- Real UI sequence.
- Outcome card.

Avoid:

- Long generated clips before the product appears.
- AI-generated dashboards as product proof.
- Hallucinated logos or signage.

## Recipe 5: Executive Explainer

Best for senior audiences with limited time.

Structure:

1. One business question.
2. One system loop.
3. One named example.
4. One proof roll-up.
5. One closing statement.

Visual actors:

- Framework loop.
- Anchor object.
- Personal example.
- Roll-up metric.

Avoid:

- Feature tours.
- Jargon.
- Small text.
- Multiple competing examples.

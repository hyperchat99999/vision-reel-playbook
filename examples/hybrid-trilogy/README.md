# Example: Hybrid Trilogy

This example structure is for a three-part product story with human setup and real UI proof.

## Fictional Scenario

Acme Workflows helps teams plan, act, and prove outcomes.

## Part 1: Plan

- Human context: a manager needs a clear plan.
- UI proof: the product creates aligned actions.
- Outcome: the team starts from the same source of truth.

## Part 2: Act

- Human context: work changes during the quarter.
- UI proof: evidence and updates flow into the plan.
- Outcome: coaching and follow-up happen earlier.

## Part 3: Prove

- Human context: leaders need confidence at year end.
- UI proof: records, evidence, and decisions reconcile.
- Outcome: outcomes are explainable and ready for review.

## Reusable Pattern

Each part should contain:

- One human setup moment.
- One real UI sequence.
- One proof artifact.
- One closing recap.

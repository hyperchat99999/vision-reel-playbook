# Contributing

Contributions should make the playbook more useful without making it less public-safe.

## Good Contributions

- New fictional examples.
- Better starter scripts.
- More QC checks.
- Improved diagrams.
- Generic production recipes.
- Public-safe before/after examples.

## Not Accepted

- Client names.
- Real company screenshots.
- Private prompts.
- API keys or credentials.
- Voice files from private work.
- Generated assets with unclear rights.
- Local machine paths.

## Before Opening A PR

Run:

```bash
npm run check
```

Also read `docs/09-ip-safety.md`.

## Style

Keep the writing direct and practical. Prefer reusable rules, examples, and checklists over long essays.
